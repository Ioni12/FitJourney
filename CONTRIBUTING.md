# Contributing to FitJourney

First off, thank you for considering contributing to FitJourney! It's people like you that make FitJourney a great tool.

## Code of Conduct
This project and everyone participating in it is governed by the FitJourney Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for FitJourney.
* **Use the provided Bug Report Template** when creating an issue.
* **Describe the bug clearly**, including steps to reproduce, what you expected to see, and what you actually saw.
* **Include environment details** (OS, browser, Node version, etc.).

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion for FitJourney.
* **Use the Feature Request Template** when creating an issue.
* **Explain why this enhancement would be useful** and to what users.

### Pull Requests
The process described here has several goals:
- Maintain FitJourney's quality
- Fix problems that are important to users
- Engage the community in working toward the best possible open source project

1. **Fork the repo** and create your branch from `main`.
2. **Branch Naming**: Use a prefix indicating the type of work (e.g., `feature/`, `bugfix/`, `hotfix/`, `docs/`) followed by a descriptive name (e.g., `feature/user-login`).
3. If you've added code that should be tested, add tests.
4. If you've changed APIs, update the documentation.
5. Ensure the test suite passes.
6. Make sure your code lints.
7. Issue that pull request using the provided PULL_REQUEST_TEMPLATE!

## Branching Strategy
We use **GitHub Flow** (or GitFlow adapted for this repository).
- `main` branch is the authoritative branch and should always be deployable.
- Feature branches are created from `main`.
- Pull Requests are required to merge code back into `main`.
