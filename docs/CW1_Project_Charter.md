# CW1 — Project Charter: FitJourney

**Project Name:** FitJourney
**Team Size:** 3-4 Students
**Project Manager / Course Contact:** [Course Participant Name]

**Problem Statement:** Many individuals struggle to maintain consistency in their fitness routines due to a lack of organized tracking tools. Existing apps are either overly complex, focusing solely on elite athletes, or too simplistic to provide meaningful insights. FitJourney aims to bridge this gap by offering a streamlined, intuitive platform that allows users to seamlessly log workouts, track nutrition, and visualize their progress over time, keeping them motivated and on track.

**Target Users:** 
1. **General Users:** Log daily workouts, track nutrition macros, view progress dashboards, and set personal fitness goals.
2. **Personal Trainers:** View assigned clients' progress, create custom workout templates, and monitor client adherence.
3. **Application Administrators (System Admins):** Manage user accounts, moderate community spaces (future phase), and generate system-wide usage reports.

**Rough Feature Scope:**
* Secure user registration, authentication, and profile management.
* Workout logging module including strength (sets, reps, weight) and cardio (duration, distance).
* Nutrition tracking module (calories, protein, carbs, fats).
* Interactive dashboard with visual analytics (graphs for weight progression and activity consistency).
* Trainer-to-client linkage for progress monitoring.

**Process Model:** We will follow the **Agile/Scrum** software development process, organizing our work into 2-week sprints. 
*Justification:* Since the requirements for a fitness application can evolve rapidly based on user feedback and changing priorities, an iterative approach guarantees that we can adapt to changes without derailing the project. Furthermore, Scrum ceremonies like Sprint Reviews will let us demonstrate incremental value to stakeholders early and often, which fits perfectly with the cyclic nature of this academic project.
