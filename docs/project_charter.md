## 2. Executive Summary
FitJourney is a fitness and wellness tracking application that empowers users to log workouts, track nutrition, and monitor their progress. This project aims to demonstrate professional software engineering and project management principles by building out features via an iterative (Agile/Scrum) development process.

## 3. Project Objectives & Scope
**Objectives:**
1. Develop a fully functional MVP for FitJourney including user authentication, workout tracking, and basic reporting.
2. Establish continuous integration and continuous deployment (CI/CD) practices.
3. Manage the project using Agile/Scrum methodologies, tracking tasks via User Stories, Sprints, and a Product Backlog.

**In Scope:**
* User Registration and Authentication
* Activity/Workout Logging module
* Dashboard for tracking progress over time
* REST API development (Backend)
* Interactive Web Interface (Frontend)

**Out of Scope:**
* Wearable device integrations (e.g., Apple Watch, Fitbit) for Phase 1
* Social networking features (e.g., user friend lists) for Phase 1

## 4. Risks & Constraints
* **Constraints:** Fixed timeline for the university course/semester.
* **Risks:** Unfamiliarity with the tech stack may cause delays; scope creep if too many features are added.

## 5. Stakeholders
* **Direct Stakeholders:** Developer Team, Project Management Course Instructor, TA
* **End Users:** Individuals looking to track their fitness visually and textually.
