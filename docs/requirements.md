# FitJourney Product Backlog

This document serves as the initial collection of User Stories and Requirements for FitJourney, categorized by Epics.

## Epic 1: User Management
**Description:** As a platform, we need secure user accounts to store personal fitness data privately.

* **User Story 1.1:** As a new user, I want to create an account using my email and password so that my data is saved securely.
* **User Story 1.2:** As a returning user, I want to log in to my account so that I can access my dashboard.
* **User Story 1.3:** As a user, I want to edit my profile (height, weight, age) so that the app gives me accurate metrics.

## Epic 2: Workout Tracking
**Description:** Core functionality to track exercises, sets, and reps.

* **User Story 2.1:** As a user, I want to log a new workout session (date, type of exercise) so that I can keep a history of my activities.
* **User Story 2.2:** As a user, I want to log specific sets, reps, and weights for strength exercises so that I can track progressive overload.
* **User Story 2.3:** As a user, I want to log duration and distance for cardio exercises so that I can track my endurance.

## Epic 3: Dashboard & Analytics
**Description:** Visualizing data to keep the user motivated.

* **User Story 3.1:** As a user, I want to view a dashboard with my recent activity history so that I know how consistent I am being.
* **User Story 3.2:** As a user, I want to see a chart of my weight progression over time so that I can see if I am reaching my goals.

## Epic 4: Nutrition Tracking (Backlog)
**Description:** Tracking caloric intake and macros.

* **User Story 4.1:** As a user, I want to log the food I eat (calories, protein, carbs, fat) so that I can meet my daily nutritional goals.
